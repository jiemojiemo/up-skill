# 录入问卷

## 任务

结构化收集 UP 主基础信息，为后续四层分析做准备。只问 3 个问题，不要问更多。

---

## 问题序列

### Q1：UP 主名/代号（必填）

```
这个 UP 主叫什么？（花名、频道名、或你给他起的代号都行）
```

用于生成 slug（全小写，空格转 `_`，去掉特殊字符）。

### Q2：基本信息（可跳过）

```
一句话描述他的基本情况：平台、内容领域、粉丝量级、性别
示例：B 站 科技区 百万粉 男
```

提取字段：
- `platform`：B 站 / YouTube / 抖音 / 小红书 / 多平台
- `domain`：科技 / 美食 / 游戏 / 知识 / 生活 / Vlog / 其他
- `followers`：万粉 / 十万粉 / 百万粉 / 千万粉
- `gender`：男 / 女 / 未知

### Q3：人设画像（可跳过）

```
一句话描述他的风格和给你的印象：
示例：毒舌 反消费主义 爱用数据说话 评论区互动很多
```

提取标签用于 Layer 0 行为规则生成。

---

## 汇总确认模板

收集完后输出：

```
好，我来确认一下：

- 名字：{name}
- 平台/领域：{platform} {domain}
- 粉丝量级：{followers}
- 风格标签：{tags}

对吗？确认后我们进入素材导入。
```

---

## Slug 生成规则

- 全小写
- 空格 → `_`
- 去掉特殊字符（括号、斜杠、点）
- 示例：`老番茄` → `lao_fan_qie`，`影视飓风` → `ying_shi_ju_feng`
